# SmartDiary 详细设计文档

## 1 产品概述

### 1.1 背景

随着社交媒体和个人记录需求的日益增长，人们越来越需要一个既能记录私人日记又能与好友分享生活点滴的平台。SmartDiary（字迹）应运而生，它是一个融合私人日记记录与社交互动功能的创新应用，满足用户在数字时代记录生活、分享心情的双重需求。

### 1.2 产品定位

SmartDiary 定位于**日记与社交的融合平台**，主要面向：
- 喜欢记录生活、分享日常的个人用户
- 希望与好友保持互动但又注重隐私保护的社交用户
- 需要私人空间记录想法、感悟的思考者

### 1.3 产品价值

**核心价值**：
- **私人日记**：提供安全、便捷的日记记录功能，支持富文本编辑
- **社交互动**：通过字迹圈功能与好友分享动态，支持点赞、评论互动
- **隐私保护**：灵活的隐私设置，用户可自主控制信息可见性
- **好友管理**：完善的好友申请、管理功能，构建安全的社交圈

## 2 产品架构

### 2.1 整体架构

SmartDiary 采用前后端分离的 B/S 架构：

```
─────────────────────────────────────────────────────────┐
│                    客户端浏览器                           │
│              (Vue3 + Element Plus)                       │
└────────────────────┬────────────────────────────────────
                     │ HTTP/REST API
────────────────────┴────────────────────────────────────┐
│              后端服务层                                   │
│          (Spring Boot + MyBatis)                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │ Controller│  │  Service │  │  Mapper  │               │
│  │   层     │  │   层     │  │   层     │               │
│  └──────────┘  └──────────┘  └──────────┘               │
└────────────────────┬────────────────────────────────────┘
                     │ JDBC
┌────────────────────┴────────────────────────────────────┐
│              数据存储层                                   │
│              (MySQL 8.0+)                                │
└─────────────────────────────────────────────────────────┘
```

### 2.2 前端展示层

**技术栈**：
- **框架**：Vue 3.5.22
- **UI 组件库**：Element Plus 2.11.8
- **路由管理**：Vue Router 4.6.3
- **HTTP 客户端**：Axios 1.13.2
- **富文本编辑器**：Quill 2.0.3
- **地址数据**：element-china-area-data 6.1.0

**页面模块**：
- **认证模块**：登录、注册、忘记密码
- **日记模块**：首页日记列表、日记详情、新建日记
- **社交模块**：字迹圈列表、字迹圈详情
- **好友模块**：好友列表、好友申请
- **设置模块**：个人设置、隐私设置

### 2.3 后端管理层

**技术栈**：
- **框架**：Spring Boot
- **ORM**：MyBatis
- **数据库**：MySQL 8.0+
- **短信服务**：阿里云 SMS

**控制器层**：
- `UserController`：用户注册、登录、资料管理
- `DiaryController`：日记 CRUD 操作
- `DiaryCircleController`：字迹圈动态管理
- `CommentController`：评论管理
- `FriendController`：好友管理

### 2.4 数据存储层

**数据库配置**：
- **数据库名**：smartdiary
- **字符集**：utf8mb4
- **存储引擎**：InnoDB
- **版本**：MySQL 8.0+

**核心数据表**：
- `user`：用户信息表
- `diary`：私人日记表
- `diary_circle`：字迹圈动态表
- `diary_circle_like`：动态点赞表
- `comment`：评论表
- `comment_like`：评论点赞表
- `friend_request`：好友申请表

SmartDiary 是一个基于 Web 的智能日记应用，提供私人日记记录和社交互动功能。系统采用前后端分离架构，前端使用 Vue3 构建用户界面，后端使用 Spring Boot 提供 RESTful API 服务，数据持久化使用 MySQL 数据库。

### 4.3 数据库设计

#### 4.3.1 数据实体

系统包含以下核心数据实体：

**用户实体 (User)**
- 基本信息：ID、UID、昵称、密码
- 联系方式：手机号、邮箱
- 个人资料：生日、地址、头像
- 隐私设置：允许搜索设置、隐藏信息设置
- 偏好设置：主题

**日记实体 (Diary)**
- 基本信息：ID、所属用户、标题、内容
- 时间信息：创建时间、更新时间

**字迹圈动态实体 (DiaryCircle)**
- 基本信息：ID、发布者、内容
- 统计信息：点赞数、评论数
- 时间信息：创建时间、更新时间

**评论实体 (Comment)**
- 基本信息：ID、所属动态、评论者、内容
- 层级关系：父评论ID
- 统计信息：点赞数
- 时间信息：创建时间、更新时间

**点赞实体 (Like)**
- 动态点赞：DiaryCircleLike（动态ID、用户ID、时间）
- 评论点赞：CommentLike（评论ID、用户ID、时间）

**好友申请实体 (FriendRequest)**
- 基本信息：ID、申请人、被申请人
- 状态：待处理、已同意、已拒绝
- 时间信息：创建时间、更新时间

#### 4.3.2 数据库表设计

详细表结构请参考 [数据库设计文档](back/docs/数据库设计文档.md)

**表关系概览**：

```
user ──┬──> diary (1:N)
       ├──> diary_circle (1:N)
       ├──> comment (1:N)
       ├──> diary_circle_like (1:N)
       ├──> comment_like (1:N)
       └──> friend_request (1:N 申请/接收)

diary_circle ──┬──> comment (1:N)
               └──> diary_circle_like (1:N)

```

## 6 核心业务流程

### 6.1 用户注册流程

```
用户填写注册信息 → 前端验证 → 后端验证 → 检查用户名是否已存在 → 
检查手机号是否已注册 → 生成UID → 密码加密 → 保存用户数据 → 
返回注册成功
```

### 6.2 日记发布流程

```
用户编写日记 → 前端富文本编辑 → 提交日记 → 后端验证 → 
保存日记内容 → 关联用户 → 记录时间 → 返回发布成功
```

### 6.3 好友申请流程

```
搜索用户 → 选择目标用户 → 发送申请 → 后端检查是否已存在申请 → 
创建申请记录 → 通知对方 → 对方处理申请（接受/拒绝）→ 
更新申请状态 → 如接受则建立好友关系
```

### 6.4 动态互动流程

```
发布动态 → 好友浏览动态 → 点赞/评论 → 更新统计计数 → 
通知动态发布者 → 其他用户可查看互动
```

## 7 安全设计

### 7.1 数据安全

- **密码安全**：当前使用 MD5 加密，生产环境建议升级为 bcrypt
- **隐私控制**：用户可自主设置搜索权限和信息可见性
- **数据隔离**：私人日记仅限创建者访问

### 7.2 接口安全

- **跨域配置**：通过 CORS 配置控制跨域访问
- **身份验证**：基于用户 ID 的会话管理
- **权限控制**：业务逻辑层面的权限验证
